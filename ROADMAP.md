# Debo Roadmap

This document outlines the planned development phases for Debo. Priorities may shift based on community feedback and technical feasibility.

---

## Phase 0 — Foundation

The core local memory inbox.

- [x] Tauri desktop app shell
- [x] Memory capture (thoughts, tasks, ideas, links, journal)
- [x] localStorage persistence
- [x] Memory timeline view
- [x] Search across memories
- [x] Task view with completion tracking
- [x] Dashboard with stats
- [x] Settings and data management
- [x] Dark/light theme support

## Phase 1 — Local Memory

Richer local-first data management.

- [ ] SQLite local database
- [ ] Import/export (JSON, Markdown)
- [ ] Tags and categories
- [ ] Backlinks between memories
- [ ] Calendar-like timeline view
- [ ] Local-first settings persistence
- [ ] Rich text editor
- [ ] File attachments
- [ ] Memory pinning and archiving

## Phase 2 — AI Layer

Intelligence that stays local.

- [ ] Summarize memories
- [ ] Auto-extract tasks from notes
- [ ] Extract people and entities
- [ ] Ask Debo — natural language queries
- [ ] Source-backed answers with citations
- [ ] Bring-your-own API key (BYOK)
- [ ] Local model support via Ollama

## Phase 3 — Desktop Power Features

Productivity features for daily use.

- [ ] Global keyboard shortcut for instant capture
- [ ] System tray app for background access
- [ ] Quick capture popup
- [ ] File import (PDF, DOCX, TXT)
- [ ] Voice notes
- [ ] Daily review and digest
- [ ] Notification reminders
- [ ] Menu bar integration

## Phase 4 — Advanced Local-First

Deep local-first capabilities.

- [ ] Local embeddings for semantic search
- [ ] Ollama integration for local LLMs
- [ ] Encrypted local storage
- [ ] Backup and restore
- [ ] Plugin system
- [ ] Custom themes
- [ ] Keyboard shortcut customization
- [ ] Multi-window support

## Phase 5 — Optional Cloud

For users who want sync, with privacy intact.

- [ ] End-to-end encrypted sync
- [ ] Paid cloud backup
- [ ] Multi-device sync
- [ ] Team and workspace features
- [ ] Shared memory collections (optional)
- [ ] Web access (encrypted, read-only)

---

## Contributing to the Roadmap

Have an idea? Open a [feature request](https://github.com/debo-life/debo-app/issues/new?template=feature_request.yml) or start a discussion.

The roadmap is a living document. Priorities shift based on what users need most.
