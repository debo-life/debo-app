# Governance

## Current Structure

Debo is currently **founder-led**.

**Maintainer:** Shaswat Raj ([@SH20RAJ](https://github.com/SH20RAJ))

The maintainer is responsible for:
- Product direction and roadmap
- Reviewing and merging pull requests
- Managing releases
- Triaging issues
- Maintaining community standards

## Decision Making

### Small changes

Bug fixes, documentation improvements, and minor UI tweaks can be reviewed and merged quickly by the maintainer. These do not require an RFC or prior discussion.

### Major features

Significant features, architectural changes, and new subsystems require:

1. An open issue describing the proposal
2. Discussion and feedback from the maintainer
3. Approval before implementation begins

For very large changes, use the [RFC template](docs/RFC_TEMPLATE.md).

### Product direction

The overall product vision and roadmap are maintained by the core maintainers. Community input is welcome and encouraged through issues and discussions.

## Contribution Ladder

Debo uses a simple contribution ladder:

### Contributor

- Anyone who has had a pull request merged
- Can open issues and submit pull requests
- Listed in contributor acknowledgments

### Regular Contributor

- Consistent, quality contributions over time
- Trusted to review others' pull requests
- May be invited to join the maintainers discussion

### Maintainer

- Full commit access to the repository
- Can merge pull requests
- Participates in roadmap and product decisions
- Added by invitation from existing maintainers

## Adding Maintainers

New maintainers are added by invitation based on:

- Consistent quality contributions
- Good judgment and communication
- Alignment with project values
- Active participation in the community

## Conflict Resolution

When disagreements arise:

1. Discussion happens in the relevant issue or PR
2. The maintainer makes a final decision if consensus is not reached
3. Decisions are documented with reasoning
4. The community is treated with respect throughout
